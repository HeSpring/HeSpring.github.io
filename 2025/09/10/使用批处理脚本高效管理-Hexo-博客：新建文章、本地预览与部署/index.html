使用批处理脚本高效管理 Hexo 博客：新建文章、本地预览与部署

发表于 2025-09-10 更新于 2025-09-10 分类于 Next 阅读次数： 本文字数： 1.9k 阅读时长 ≈ 6 分钟

前言

在日常使用 Hexo 进行博客写作和管理时，我们常常需要重复执行一些命令：新建文章、生成和预览网站、部署到远程仓库等。为了提高效率，可以通过 批处理脚本（.bat） 来自动化这些操作。本文将分享我制作的三个实用脚本，并详细说明其功能和使用方法。

一、新建文章脚本

脚本功能

这个脚本用于快速在 Hexo 博客中创建新文章，并自动打开文章文件以便立即编辑。具体功能包括：

  • 自动切换到 Hexo 博客根目录
  • 提示输入文章标题
  • 调用 hexo new 创建新文章
  • 自动生成文件路径（空格自动转换为短横线）
  • 检查文件是否存在，若存在则自动打开，否则打开 _posts 文件夹以便手动查找

脚本示例

使用方法

  1. 将脚本保存为 new_post.bat
  2. 双击运行，输入文章标题
  3. 脚本会自动创建文章并打开编辑器

二、本地预览网站脚本

脚本功能

在撰写或修改博客时，常常需要本地生成并预览网站效果。该脚本实现：

  • 自动生成 Hexo 静态文件
  • 启动本地服务器（默认端口 4000）
  • 自动打开浏览器预览

脚本示例

使用方法

  1. 保存为 preview.bat
  2. 双击运行，浏览器会自动打开本地博客
  3. 关闭命令窗口即可停止本地服务器

三、部署网站脚本

脚本功能

当博客内容修改完成后，需要部署到远程仓库。此脚本实现：

  • 清理旧生成文件
  • 重新生成静态文件
  • 调用 hexo deploy 上传到 GitHub Pages 或其他远程服务

脚本示例

使用方法

  1. 保存为 deploy.bat
  2. 双击运行，脚本会自动完成部署

总结

通过这三个批处理脚本，你可以：

  1. 快速新建文章并立即编辑
  2. 一键生成网站并本地预览
  3. 自动部署博客到远程仓库

这大大简化了 Hexo 博客的日常操作，提高了写作和管理效率。如果你也在使用 Hexo，强烈推荐尝试这些自动化脚本。
