前言

在国内网络环境下，很多人使用 Hexo 部署博客到 GitHub Pages 时会遇到一个问题：使用 SSH 默认的 22 端口会无法连接 GitHub，报错类似：

ssh: connect to host github.com port 22: Connection timed out

即使挂了 VPN，也无法通过 22 端口访问 GitHub 的 SSH 服务。

解决思路

GitHub 支持通过 443 端口的 SSH 服务，这样可以绕过国内网络对 22 端口的屏蔽。
通过修改 SSH 命令和 Hexo 部署配置，就能成功部署博客。

总结

  1. 国内网络环境下，SSH 22 端口经常被屏蔽，导致 Hexo 部署失败。
  2. 使用 GitHub 提供的 SSH 443 端口 可以解决这一问题。
  3. Hexo 配置 repo 改为 ssh://git@ssh.github.com:443/... 即可。
  4. 挂 VPN 并非必需，只要 443 端口通即可。

这样，你就可以顺利通过 SSH 部署 Hexo博客，无需切换到 HTTPS 或 Formspree 等方式。

💡 Tip：如果你不想配置 SSH，也可以直接使用 HTTPS 部署，但每次推送需要输入 Personal Access Token。
